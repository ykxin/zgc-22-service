# AI家政双向信任匹配平台

> **Vue3 + FastAPI 全栈项目 | 前后端分离 | AI智能匹配 | 开箱即用**

## 一、项目简介

AI家政双向信任匹配平台是一套完整的家政服务数字化解决方案，采用 **Vue3 + Element Plus** 前端和 **FastAPI + SQLAlchemy** 后端技术栈，集成 **AI资质核验、AI服务标准量化、AI双向智能匹配、AI纠纷仲裁** 四大核心智能模块，实现雇主与从业者之间的双向信任匹配。

### 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Vue 3.4 + Vite 5 | 组合式API + 响应式 |
| UI组件库 | Element Plus 2.4 | 统一风格 |
| 状态管理 | Pinia 2 | 轻量级 |
| HTTP 客户端 | Axios 1.6 | 拦截器封装 |
| 后端框架 | FastAPI 0.104 | 异步高性能 |
| ORM | SQLAlchemy 2.0 | 数据库操作 |
| 数据库 | SQLite | 零配置 |
| 认证 | JWT + bcrypt | 令牌认证 |
| 数据校验 | Pydantic 2.5 | 类型安全 |

## 四、功能模块详解

### （一）用户认证模块

- 雇主注册/登录（手机号 + 密码）
- 从业者注册/登录（手机号 + 短信验证码 + 密码）
- JWT令牌生成与验证（24小时有效期）
- 角色权限控制（employer / worker / admin）
- 个人信息修改（含画像更新）
- 密码重置

### （二）AI资质智能核验模块

```
从业者上传证件图片 → AI OCR识别 → 信息校验 → 信用评分生成
```

- **证件OCR识别**：身份证（姓名、号码、性别、有效期）、健康证、资格证
- **真伪校验**：OCR置信度检查、有效期判断、证件号格式校验、重复注册校验
- **信用评分算法**（满分100分）：
  - 资质完整度：30分（每证10分）
  - 服务完成率：25分
  - 评价均分：25分
  - 纠纷记录：20分
  - 有效期加分：每有效证+3分
- **风险等级**：≥80低风险 / ≥60中风险 / <60高风险
- **资质状态**：待审核 → 已通过 · 已驳回 · 已过期
- **到期提醒**：30天内到期证件自动提醒

### （三）AI服务标准量化模块

平台内置3大类SOP标准（共19个细分步骤）：

| 类别 | 步骤数 | 包含步骤 |
|------|--------|----------|
| 保洁 | 6步 | 除尘打扫、地面清洁、厨房清洁、卫浴清洁、死角清理、垃圾分类 |
| 育儿 | 6步 | 晨间护理、教育活动、户外活动、午餐午休、下午活动、晚间交接 |
| 养老陪护 | 7步 | 晨间护理、用药管理、膳食准备、康复训练、陪伴交流、安全巡查、晚间护理 |

- 从业者按SOP步骤逐项打卡（含图片上传）
- AI自动验收打分（步骤完成度 + 图片核验加分 + 自定义标准加权）
- 生成验收报告（总分、等级、步骤详情）
- 雇主可自定义服务标准（增删要求、调整权重）

### （四）AI双向智能匹配模块

**匹配算法**（5维度加权计算）：

| 维度 | 满分 | 计算方法 |
|------|------|----------|
| 信用分匹配 | 25分 | 从业者信用分/100 × 25 |
| 技能匹配度 | 30分 | Jaccard相似度（雇主需求 ∩ 从业者技能 / 并集） |
| 距离适配度 | 15分 | Haversine公式，<3km满分，>20km递减 |
| 时间适配度 | 15分 | 服务时段与雇主作息的重叠度 |
| 场景适配度 | 15分 | 特殊需求匹配（老人/小孩/宠物） |

- 为雇主生成 **Top5 推荐列表**，展示匹配得分详情
- 雇主查看推荐、发起预约
- 从业者接收/拒绝预约

### （五）AI纠纷公允判定模块

```
纠纷提交 → 自动取证 → AI责任判定 → 生成仲裁报告
```

- **自动取证**：关联订单信息、验收报告、打卡记录、聊天记录、评价记录
- **责任判定规则**（分3类纠纷）：
  - **服务质量**：验收分<60 → 从业者责任；≥80 → 雇主责任
  - **薪资争议**：验收分≥70 → 雇主责任；<70 → 双方协商
  - **物品损耗**：聊天记录有证据 → 从业者责任；无证据 → 双方协商
- **处理建议**：退款、补服务、撤销差评、协商调解
- **状态跟踪**：待处理 → 已判定 → 已执行

### （六）订单与评价模块

- **订单全流程**：预约 → 接单 → 服务中 → 验收 → 完成 → 评价
- **双向评价**：雇主评从业者 · 从业者评雇主
- **评分联动**：评价直接影响从业者信用分，影响后续匹配

### （七）管理后台模块

- **数据面板**：用户数、订单量、匹配成功率、纠纷率、类别分布
- **用户管理**：查看/禁用/启用雇主和从业者账号
- **资质审核**：通过/驳回从业者资质
- **订单管理**：查看全部订单及纠纷
- **SOP管理**：新增/编辑/上下架服务标准步骤

## 六、快速启动

### 环境要求

- Python 3.9+
- Node.js 16+
- pip / npm

### 1. 启动后端

```bash
cd backend

# 安装依赖
pip install -r requirements.txt

# 启动服务（自动建表 + 初始化管理员 + 默认SOP数据）
python main.py
# 或：uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

服务地址：`http://localhost:8000`

### 2. 启动前端

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端地址：`http://localhost:3000`（已配置代理到后端8000端口）

## 八、AI算法说明

### 信用评分算法

```
总分 = 资质完整度(30) + 完成率(25) + 评价均分(25) + 纠纷扣分(20) + 有效期加分
风险等级：≥80低 / ≥60中 / <60高
```

### 双向匹配算法

```
匹配分 = 信用分(25) + 技能Jaccard(30) + Haversine距离(15) + 时段重叠(15) + 场景适配(15)
结果：按匹配分降序取Top5
```

### 纠纷判定规则

| 纠纷类型 | 从业者责任条件 | 雇主责任条件 | 默认 |
|----------|---------------|-------------|------|
| 服务质量 | 验收分 < 60 | 验收分 ≥ 80 | 双方 |
| 薪资 | 验收分 < 70 | 验收分 ≥ 70 | 双方 |
| 物品损耗 | 聊天记录有证据 | - | 双方 |

### 验收打分算法

```
步骤得分 = SOP默认分 × (完成?1:0) × (有图片?1.1:1)
总分 = Σ步骤得分 / Σ步骤满分 × 100 + 自定义标准加减分
等级：≥90优秀 ≥80良好 ≥60合格 <60不合格
```
